Analytical Reasoning (Logic Games) Removed Aug 2024

Historical reference — this section no longer appears on the LSAT. Previously: one scored section with 4 games (~23 questions total).

The Universal LG Process

  1. Read the scenario — identify game type, players, and slots.
  2. Set up your master diagram — draw slots/groups before touching rules.
  3. Symbolize every rule clearly and write the contrapositive where applicable.
  4. Make inferences by combining rules before answering any questions.
  5. Answer "If" questions in a fresh diagram, not the master.
  6. Use elimination — work with the answer choices rather than building from scratch on every question.

Game Type Reference

Linear / Sequencing

Place variables into ordered slots (positions 1–7, Monday–Friday, etc.). The most common game type on the LSAT.

Example Setup Seven houses on a street — A, B, C, D, E, F, G — each in a unique position from 1 to 7.

Diagram Approach

  1. Draw 7 numbered slots in a row
  2. List variables to the side: A B C D E F G
  3. Symbolize ordering rules: A < B (A before B)
  4. Use dashes for blocks: [AB] (A immediately before B)

Grouping

Assign variables into two or more distinct groups. Focus on who can/cannot be in the same group.

Example Setup Six employees split into two teams of three: Team 1 and Team 2.

Diagram Approach

  1. Draw two columns with labeled headers
  2. Note group sizes and any fixed assignments
  3. Symbolize "not together": A ≠ B (A and B in different groups)
  4. Symbolize "together": A = B (must be in same group)

In/Out (Selection)

Select a subset of variables to be "in" while the rest are "out." Conditional rules dominate.

Example Setup A committee selects at least 3 of 7 candidates: A, B, C, D, E, F, G.

Diagram Approach

  1. Draw two columns: IN | OUT
  2. Translate all rules into conditional form
  3. Always write contrapositive next to each rule
  4. Chain conditionals together to find forced outcomes

Matching

Assign attributes to variables or vice versa. Variables can receive multiple attributes.

Example Setup Five students each take one or more of three subjects: Math, Science, English.

Diagram Approach

  1. Draw a grid: variables as rows, attributes as columns
  2. Fill in cells as Yes / No / Unknown
  3. Look for forced placements from rules

Hybrid Games

Combines two game types — usually sequencing + grouping or selection + sequencing. More complex but same tools apply.

Example Setup Assign 6 people to 3 ranked positions (1st, 2nd, 3rd) with exactly 2 people per position.

Diagram Approach

  1. Identify both components of the hybrid
  2. Build a diagram that captures both dimensions
  3. Separate rules by type; apply each component's rules methodically

Advanced Linear (Multi-Tier)

Two rows of ordered slots that must be matched to each other (e.g., employees scheduled to floors on days).

Example Setup Four people visit on Monday–Thursday; each is assigned either morning or afternoon.

Diagram Approach

  1. Stack two rows with shared column headers (days/positions)
  2. Row 1 = one attribute; Row 2 = another
  3. Rules often link values in different rows

Rule Symbolization Quick Reference

Rule WordingSymbolContrapositive / Implication
"A is before B" A < B B cannot be before A; if B = slot 1, A cannot be placed
"A is immediately before B" AB (block) A cannot be in last slot; B cannot be in first slot
"A is not adjacent to B" A ≁ B No slot where |pos(A) − pos(B)| = 1
"If A is selected, B is selected" A → B ¬B → ¬A
"A and B cannot both be selected" ¬(A ∧ B) A → ¬B and B → ¬A
"At least one of A or B is selected" A ∨ B ¬A → B and ¬B → A
"Exactly one of A, B, C is selected" 1 of {A,B,C} If A is in, B and C are out; etc.
"A is in the same group as B" A ↔ B If A is in Group 1, B is in Group 1; split contrapositively

Logic Games Question Types

TypeStem ClueBest Approach
Complete & Accurate List "Which could be a complete and accurate list of..." Use each rule to eliminate one answer choice. Fastest when done in order.
Could Be True "Which of the following could be true?" Find the answer that doesn't violate any rule. Eliminate 4 answers that must be false.
Must Be True "Which of the following must be true?" Eliminate answers that are only sometimes true. The correct answer holds in all valid scenarios.
Cannot Be True "Which of the following cannot be true?" Find the answer that violates at least one rule in every scenario.
Conditional "If" Question "If A is placed in position 3, which..." Draw a fresh diagram, add the new condition, make all inferences, then answer.
Rule Substitution "Which rule could replace [rule X] while having the same effect?" Re-derive what the original rule did (its net effect) and find the answer that produces the same constraint.
Add a Rule "If the following additional rule is in effect..." Treat like a conditional "If" question — add to master diagram and make inferences.

High-Value Tips

Work "If" Questions First

Conditional questions give you free information. The work you do builds towards the global questions too.

Identify "Floaters"

Variables with no rules are floaters — they can often go anywhere. Track them separately to avoid overlooking them.

Build a Limited Scenario Template

If a game has only 2–3 ways things can be arranged, sketch all scenarios upfront and answer questions by checking templates.

Never Skip Inferences

Rules that share a variable produce inferences. Combine every pair of rules that share a variable. This is where speed comes from.

Watch Game Size / Slot Count

If you have 7 variables and 7 slots, it's a 1:1 placement. If there are 8 variables and 5 slots, some must be out — a selection component is implied.

Time Budget: 8–9 min per game

With 35 minutes for 4 games, aim for ~8 minutes each. Harder games often come 2nd or 3rd — skip and return if stuck.
